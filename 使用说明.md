# VRCTranslate 使用说明

> 平台：Windows 10/11 64 位、VRChat PC 桌面模式  
> 环境安装和启动命令：[README.md](README.md)

## 界面语言

设置页顶部有语言按钮，点击可在中文、English、日本語之间循环切换，切换后立即保存并刷新全部界面文字，无需重启。

## 1. 使用前准备

1. 启动 VRChat 桌面模式。
2. 在 VRChat 快捷菜单中启用 OSC。
3. 启动 VRCTranslate。
4. 在“设置 → 翻译模型 → 服务档案”建立翻译档案，再到“翻译路由”分别选择自身消息和 OCR 服务。

程序启动后会出现主窗口和单行快捷输入浮窗。主窗口只负责配置与 OCR 控制，永不强制置顶。

## 2. 快捷输入自己的文字

输入浮窗为空时显示：

> 按Enter发送消息至VRChat

操作方法：

1. 在浮窗中输入文字。
2. 按 Enter。
3. 程序按“自身消息服务”翻译。
4. 翻译成功后按设置格式处理，并自动加入 OSC 发送队列。

浮窗没有翻译、清空、发送按钮，也没有可编辑译文框。翻译成功后，浮窗卡片会显示最近一条的原文和译文预览，方便确认发送内容。翻译失败时原文会恢复到输入框，预览区保持上一条成功结果；消息过长时按设置执行自动拆分、截断或拒绝发送，默认是自动拆分。

### 输入状态

OSC typing 同步始终开启：

- 输入第一个字符时发送 `typing=true`。
- 每次按键刷新活动计时。
- 停止输入约 1.5 秒、按 Enter、清空、隐藏浮窗或退出时发送 `typing=false`。

VRChat 必须已启用 OSC。OSC 使用 UDP，软件只能确认数据已发往本机端口，无法确认 VRChat 最终是否显示消息或 `...`。

### 浮窗操作

- 在主窗口“快捷输入 → 快捷输入浮窗设置”控制置顶和宽度，修改后自动保存。
- 可拖动浮窗边缘空白处改变位置。
- 关闭浮窗只会隐藏；可通过托盘菜单或双击托盘图标再次显示。
- 托盘图标双击默认打开快捷输入。

## 3. 翻译服务档案

每个档案保存一种服务的参数。自身消息和 OCR 可以选择不同档案，互不覆盖。

### 测试回显

不执行真实翻译，只返回带标记的原文。适合验证 UI 和 OSC。

### DeepL API

1. 新建或选择档案。
2. 服务类型选择“DeepL API”。
3. 填写 API 密钥。
4. Free 用户可把接口地址留空；其他端点按服务商文档填写。
5. 点击“使用固定短句测试翻译”，成功后保存。

DeepL 延迟低、质量高，适合 OCR。

### Google Cloud Translation

1. 在 Google Cloud 启用 Cloud Translation API 并准备 API 密钥。
2. 服务类型选择“Google Cloud Translation”。
3. 填写 API 密钥；官方默认地址可留空。
4. 测试后保存。

Google 语言覆盖广、延迟低，也适合 OCR。

### Google 翻译（免费）

使用 Google 翻译官方免费接口，无需 API 密钥。在服务类型中选择"Google 翻译（免费）"即可使用。

注意：国内网络通常无法直连 Google，需要代理才能使用。接口地址可留空（默认使用 translate.googleapis.com），如有可用镜像可填入接口地址字段。

### 腾讯云翻译

使用腾讯云机器翻译 TMT 的文本翻译接口。

1. 在腾讯云控制台开通机器翻译服务。
2. 获取 SecretId 和 SecretKey。
3. 服务类型选择"腾讯云翻译"。
4. API 密钥填入 SecretId，模型字段填入 SecretKey。
5. 接口地址可留空（默认使用 tmt.tencentcloudapi.com）。
6. 测试后保存。

腾讯翻译限制 5 次/秒，不支持自动识别源语言，需在翻译路由中明确指定源语言。单次文本需低于 6000 字符。每月有免费额度，超出后按量计费。

### OpenAI 兼容接口

填写兼容 API 根地址、API 密钥和模型名称。此模式更适合自身消息和口语翻译；用于 OCR 时界面会提示延迟风险，并自动限制为单并发、最多两条排队和过期结果丢弃。

## 4. 路由与发送设置

“设置 → 翻译模型 → 翻译路由”包含两组互不依赖的选择，分别放在独立卡片中：

- 自身消息（OSC）：服务、源语言、目标语言、消息格式、超长消息策略、日语罗马音自动转假名。
- 他人消息（OCR）：服务、源语言、目标语言、日语罗马音自动转假名。

### 日语罗马音自动转假名

当源语言为日语或自动检测时，开启此功能后，输入的罗马音（如 "konnichiwa"、"arigatou"）会自动转换为平假名（こんにちは、ありがとう）后再送入翻译引擎，解决日语罗马音无法直接翻译的问题。

- OSC 和 OCR 各自独立开关，互不影响。
- 仅对纯拉丁字母文本生效，包含假名或汉字的文本不会被转换。
- 自动检测模式下，会先判断文本是否像日语罗马音再决定是否转换。

自身消息还可设置：

| 项目        | 说明                               |
| ----------- | ---------------------------------- |
| 仅译文      | 只发送翻译结果                     |
| 原文 / 译文 | 同时发送原文和译文                 |
| 译文 / 原文 | 先译文后原文                       |
| 自动拆分    | 超长时按 UTF-16 长度分为多条，推荐 |
| 自动截断    | 只发送限制内部分                   |
| 拒绝发送    | 恢复原文并提示                     |

OSC 默认主机 `127.0.0.1`、端口 `9000`、最小间隔 1.5 秒。普通本机使用通常无需修改。

## 5. OCR 翻译其他玩家文字

VRChat OSC 不提供其他玩家的 Chatbox 原文，因此此功能使用窗口 OCR。

OCR 识别语言由「设置 → 翻译模型 → 翻译路由 → OCR 源语言」决定。源语言设为日语或自动检测时使用日语识别模型（可正确识别中、日、英及数字），明确选择中文等语言时使用中文模型。

### 选择识别区域

1. 保持 VRChat 窗口可见。
2. 进入“主窗口 → OCR 翻译”。
3. OCR 悬浮球会随程序默认显示，不需要手动打开。
4. 在 OCR 页“目标程序”中选择 VRChat；列表不完整时点击“刷新”。
5. 右键悬浮球选择“框选识别区域”，在 VRChat 文字经常出现的位置拖动；按 Esc 可取消。

区域越小，识别越快且误识别越少。

框选后会出现透明的 OCR 识别框。拖动顶部控制条可移动，拖动边缘或四角可缩放；右上角 `X` 会停止 OCR 并移除当前区域，但保留悬浮球以便重新框选。识别框顶部只有“单次”和“持续”两个模式入口。

选区会从 Qt 界面坐标换算到 VRChat 客户区坐标，可处理常见的 100%、125%、150%、175% 和 200% Windows 缩放。选择完成后如果移动或缩放 VRChat、改变系统缩放、旋转屏幕，或把窗口移动到不同缩放比例的显示器，请重新选择区域。VRChat 窗口不要横跨两个显示器。

### 启动 OCR

单击悬浮球可按当前模式开始或暂停，也可在识别框或悬浮球右键菜单中直接选择模式。OCR 主页面的状态、模式、语言和翻译服务仅用于展示；目标程序可以在 OCR 页面直接选择，语言和翻译服务仍在“设置 → 翻译模型 → 翻译路由”中修改。

- 单次识别：只捕获、识别和翻译一次，完成后自动回到空闲，识别框继续保留。
- 持续识别：按识别间隔取帧，结合画面变化阈值决定是否执行 OCR；同一识别框中静止区域的文字不会因为其他区域变化而反复翻译。

默认的“自动”模式优先使用 Windows Graphics Capture 获取所选窗口内容。一次 OCR 运行只建立一个持续捕获会话，不会为每一帧反复创建原生捕获对象。普通窗口覆盖在 VRChat 上方时，不应被 OCR 当成 VRChat 画面。目标窗口列表会排除 VRCTranslate 自己的进程，并同时显示窗口标题和进程名。

可在“设置 → OCR → 捕获与识别”选择后端并点击“测试捕获（仅内存预览）”。预览会在设置页显示，切换目标、关闭页面或退出后释放，不会生成图片文件。

只有在窗口捕获不兼容时才选择“MSS 屏幕坐标兼容模式”。该模式截取桌面上当前可见像素，VRChat 必须保持可见且不能被其他窗口遮挡。程序不再通过周期性隐藏和显示自身窗口完成捕获；无法安全排除时会给出错误提示。

译文浮窗默认同时显示较小的识别原文和醒目的译文；可在 OCR 主页面关闭“显示识别原文”，此时浮窗只保留译文。浮窗不显示时间、置信度或 OCR 控件。停止 OCR 后旧会话中尚未返回的译文会被忽略，已经显示的结果按显示时长自然消失。

### 低延迟规则

- 传统在线服务最多并发 2，大模型使用单并发。
- 队列达到设置上限时直接丢弃新任务，不无限积压。
- 任务超过有效期后不再展示。
- 同一会话的相同文字使用内存缓存，退出后清空。
- 持续模式按文字框位置保存上一轮内存快照，只提交新增或改变区域的文字，不使用“重复冷却秒数”。
- 结果按识别顺序展示，不因网络返回顺序而错位。

### 译文浮窗

在“主窗口 → OCR 翻译 → OCR 译文浮窗与位置尺寸”中可调整：

- OCR 译文置顶。
- 锁定并鼠标穿透。
- 是否显示识别原文。
- 背景透明度、字号、最大显示条数和单条显示时长。透明度只改变浮窗背景，文字保持清晰。
- 原文、译文和拖动条始终带有独立的深色高对比度衬底；即使外框透明度较低，在白色或彩色游戏画面上也不会直接使用浅色文字裸显。
- “显示译文浮窗”可在没有新识别结果时检查外观；“重置位置和尺寸”会恢复默认大小并放回主屏幕右上区域。下方摘要会显示当前坐标和尺寸。

需要移动或缩放时，先关闭鼠标穿透：按住浮窗顶部专用拖动条移动，从右下角拖动缩放，完成后再锁定。位置、尺寸和 OCR 页面中的浮窗设置都会自动保存。MSS 模式仍建议把浮窗移出识别区。

主窗口不会从 Windows 截图工具中排除，因此正常截图时应当可见。快捷输入和 OCR 译文浮窗仍可能使用系统捕获排除，以避免 MSS 模式重复识别自身内容。

## 6. OCR 隐私与限制

- 捕获帧只在内存中短暂处理，没有截图保存功能或图片写盘路径。
- 画面不会上传；在线翻译接口只收到 OCR 得到的文字。
- 识别原文、译文和会话缓存退出后清除。
- 不建立聊天数据库或历史记录。
- 特殊字体、小字号、透明文字、动态背景和遮挡可能降低准确率。
- Windows 窗口捕获通常不受普通窗口遮挡影响，但窗口最小化、受保护内容或独占全屏仍可能无法返回有效帧；建议使用无边框窗口。

## 7. 便携文件与日志

所有可写数据位于软件目录的 `data`：

| 路径                | 内容                          |
| ------------------- | ----------------------------- |
| `data\config.json`  | 版本 3 设置和明文 API 密钥    |
| `data\logs\app.log` | 轮转状态日志，最多约 4 个文件 |
| `data\cache`        | 运行与构建缓存                 |

日志只记录状态、请求编号和错误类别，不记录聊天正文、API 密钥或截图。在“设置 → 便携数据与诊断”可清空运行日志。

移动或备份软件时应处理整个软件目录。删除整个目录即可同时移除配置和缓存。

## 8. 常见问题

### 按 Enter 后 VRChat 没有文字

检查 VRChat 是否启用 OSC、主机是否为 `127.0.0.1`、端口是否为 `9000`，并等待最小发送间隔。程序提示“已提交至 OSC”不等于 VRChat 已显示。

### VRChat 没有出现 `...`

确认 VRChat OSC 已启用并允许 Chatbox。typing 同步在应用中始终开启，不存在设置开关；UDP 不提供回执。

### OCR 翻译太慢

优先给 OCR 选择 DeepL、Google 或腾讯翻译，不建议使用大模型；减小识别区域，并按电脑性能适当增加识别间隔。

### 软件提示 data 不可写

把完整软件目录移动到 D/E 盘普通目录后再运行。不要放在 `Program Files`，不要直接从压缩包内部启动。程序不会改用 C 盘 AppData。

### 找不到 VRChat 窗口或没有识别结果

在 OCR 页重新选择进程名为 VRChat 的目标程序，再通过悬浮球框选更精确的文字区域，然后到“设置 → OCR”执行内存测试捕获。Windows 捕获失败时查看明确的后端提示；MSS 模式则必须保持 VRChat 可见且不被遮挡。两个程序尽量使用相同权限级别。

### OCR 运行时出现标题为 VRCTranslate 的空白窗口

问题根因是旧实现先对尚未设置父控件的“识别原文”标签调用显示，再把它加入译文卡片布局。Qt 会在这段极短时间内把标签当作普通顶层窗口，因此出现标题为 `VRCTranslate` 的空白框。当前版本在创建标签时立即指定译文卡片为父控件，从源头取消了临时顶层窗口，不依赖窗口隐藏或拦截守卫。更新源码后必须完全退出旧进程再重新启动。

## 9. 安全边界

- 不要分享 `data\config.json`。
- 软件只支持 PC 桌面模式，不支持 PCVR。
- OCR 是屏幕辅助识别，不是直接读取 VRChat 聊天数据。
- 机器翻译和 OCR 都可能出错，重要内容应结合原画面判断。
