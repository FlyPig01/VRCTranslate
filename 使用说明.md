# VRCTranslate 使用说明

> 平台：Windows 10/11 64 位、VRChat PC 桌面模式  
> 环境安装和启动命令：[README.md](README.md)

## 界面语言

设置页顶部有语言按钮，点击可在中文、English、日本語之间循环切换，切换后立即保存并刷新全部界面文字，无需重启。

## 1. 使用前准备

1. 启动 VRChat 桌面模式。
2. 在 VRChat 快捷菜单中启用 OSC。
3. 启动 VRCTranslate。
4. 在“设置 → 翻译模型 → 服务档案”建立翻译档案，再到“翻译路由”分别选择自身消息和 OCR 服务。

程序启动后会出现主窗口和单行快捷输入浮窗。主窗口只负责配置与 OCR 控制，永不强制置顶。

## 2. 快捷输入自己的文字

输入浮窗为空时显示：

> 按Enter发送消息至VRChat

操作方法：

1. 在浮窗中输入文字。
2. 按 Enter。
3. 程序按“自身消息服务”翻译。
4. 翻译成功后按设置格式处理，并自动加入 OSC 发送队列。

浮窗没有翻译、清空、发送按钮，也没有可编辑译文框。翻译成功后，浮窗卡片会显示最近一条的原文和译文预览，方便确认发送内容。翻译失败时原文会恢复到输入框，预览区保持上一条成功结果；消息过长时按设置执行自动拆分、截断或拒绝发送，默认是自动拆分。

### 输入状态

OSC typing 同步始终开启：

- 输入第一个字符时发送 `typing=true`。
- 每次按键刷新活动计时。
- 停止输入约 1.5 秒、按 Enter、清空、隐藏浮窗或退出时发送 `typing=false`。

VRChat 必须已启用 OSC。OSC 使用 UDP，软件只能确认数据已发往本机端口，无法确认 VRChat 最终是否显示消息或 `...`。

### 浮窗操作

- 在“设置 → OSC → 快捷输入浮窗”控制快捷输入置顶和宽度。
- 可拖动浮窗边缘空白处改变位置。
- 关闭浮窗只会隐藏；可在主窗口“快捷输入”页或托盘菜单再次显示。
- 托盘图标双击默认打开快捷输入。

## 3. 翻译服务档案

每个档案保存一种服务的参数。自身消息和 OCR 可以选择不同档案，互不覆盖。

### 测试回显

不执行真实翻译，只返回带标记的原文。适合验证 UI 和 OSC。

### DeepL API

1. 新建或选择档案。
2. 服务类型选择“DeepL API”。
3. 填写 API 密钥。
4. Free 用户可把接口地址留空；其他端点按服务商文档填写。
5. 点击“使用固定短句测试翻译”，成功后保存。

DeepL 延迟低、质量高，适合 OCR。

### Google Cloud Translation

1. 在 Google Cloud 启用 Cloud Translation API 并准备 API 密钥。
2. 服务类型选择“Google Cloud Translation”。
3. 填写 API 密钥；官方默认地址可留空。
4. 测试后保存。

Google 语言覆盖广、延迟低，也适合 OCR。

### Google 翻译（免费）

使用 Google 翻译官方免费接口，无需 API 密钥。在服务类型中选择"Google 翻译（免费）"即可使用。

注意：国内网络通常无法直连 Google，需要代理才能使用。接口地址可留空（默认使用 translate.googleapis.com），如有可用镜像可填入接口地址字段。

### 腾讯云翻译

使用腾讯云机器翻译 TMT 的文本翻译接口。

1. 在腾讯云控制台开通机器翻译服务。
2. 获取 SecretId 和 SecretKey。
3. 服务类型选择"腾讯云翻译"。
4. API 密钥填入 SecretId，模型字段填入 SecretKey。
5. 接口地址可留空（默认使用 tmt.tencentcloudapi.com）。
6. 测试后保存。

腾讯翻译限制 5 次/秒，不支持自动识别源语言，需在翻译路由中明确指定源语言。单次文本需低于 6000 字符。每月有免费额度，超出后按量计费。

### OpenAI 兼容接口

填写兼容 API 根地址、API 密钥和模型名称。此模式更适合自身消息和口语翻译；用于 OCR 时界面会提示延迟风险，并自动限制为单并发、最多两条排队和过期结果丢弃。

### Argos Translate

Argos 是本地离线翻译，质量取决于语言模型。普通发布版可能不包含组件；离线发布版或安装 `offline` 可选依赖后才可使用。离线翻译功能已内置分句处理，不依赖网络。

在“设置 → 翻译模型 → Argos 本地模型”中：

1. 页面先检查 Argos 组件和便携模型索引；没有本地索引时会在后台尝试更新一次。索引下载通过全球 CDN 镜像加速，国内外均可正常访问。
2. 在“原文语言（从）”选择玩家文字语言，在“译文语言（到）”选择需要翻译成的语言。
3. 确认“将安装的模型”和下方安装摘要。界面同时显示中文名称与 Argos 官方代码，例如“英语（en）→ 中文（zh）”。
4. 索引失败时按页面原因重试；成功后选择语言对并点击“安装所选模型”。安装时显示实时下载进度条（已下载 / 总大小 / 百分比）。
5. 安装完成后建立 Argos 档案，并选择到自身或 OCR 路由。

模型下载临时文件位于 `data\cache\downloads`，成功安装后删除；模型位于 `data\models\argos`。页面可查看占用、删除模型或打开目录。应用不会在运行时调用 pip。

Argos 的自动语言判断主要优化中文、日语、韩语、俄语和常见英文；其他语言建议在路由中明确选择源语言。

#### 中转翻译

Argos 官方索引不包含所有语言对的直接模型（例如没有中文 ↔ 日语、中文 ↔ 韩语的直接模型）。当没有直接模型时，软件会自动通过已安装的中间语言进行中转翻译（例如：中文 → 英语 → 日语）。

- 在 Argos 模型页选择没有直接模型的语言对时，页面会提示可用的中转方案，并显示“安装中转所需模型”按钮，一键安装所需的全部模型。
- 在翻译路由页选择 Argos 档案且当前语言对没有直接模型时，OSC 和 OCR 卡片会分别显示中转警告："此语言对将通过英语中转翻译，极不可靠，可能出现严重语义偏差或幻觉内容，重要沟通请使用在线翻译服务"。
- 中转翻译经过两次翻译，极不可靠，可能出现严重语义偏差或幻觉内容。重要沟通请使用在线翻译服务。

## 4. 路由与发送设置

“设置 → 翻译模型 → 翻译路由”包含两组互不依赖的选择，分别放在独立卡片中：

- 自身消息（OSC）：服务、源语言、目标语言、消息格式、超长消息策略、日语罗马音自动转假名。
- 他人消息（OCR）：服务、源语言、目标语言、日语罗马音自动转假名。

当选择 Argos 档案且语言对没有直接模型时，对应卡片会显示中转翻译提示或缺少模型的警告。

### 日语罗马音自动转假名

当源语言为日语或自动检测时，开启此功能后，输入的罗马音（如 "konnichiwa"、"arigatou"）会自动转换为平假名（こんにちは、ありがとう）后再送入翻译引擎，解决日语罗马音无法直接翻译的问题。

- OSC 和 OCR 各自独立开关，互不影响。
- 仅对纯拉丁字母文本生效，包含假名或汉字的文本不会被转换。
- 自动检测模式下，会先判断文本是否像日语罗马音再决定是否转换。

自身消息还可设置：

| 项目        | 说明                               |
| ----------- | ---------------------------------- |
| 仅译文      | 只发送翻译结果                     |
| 原文 / 译文 | 同时发送原文和译文                 |
| 译文 / 原文 | 先译文后原文                       |
| 自动拆分    | 超长时按 UTF-16 长度分为多条，推荐 |
| 自动截断    | 只发送限制内部分                   |
| 拒绝发送    | 恢复原文并提示                     |

OSC 默认主机 `127.0.0.1`、端口 `9000`、最小间隔 1.5 秒。普通本机使用通常无需修改。

## 5. OCR 翻译其他玩家文字

VRChat OSC 不提供其他玩家的 Chatbox 原文，因此此功能使用窗口 OCR。

### 选择识别区域

1. 保持 VRChat 窗口可见。
2. 进入“主窗口 → OCR 翻译”。
3. 点击“刷新窗口”并选择 VRChat。
4. 点击“选择识别区域”。
5. 主窗口隐藏后，在 VRChat 文字经常出现的位置拖动框选；按 Esc 可取消。

区域越小，识别越快且误识别越少。

选区会从 Qt 界面坐标换算到 VRChat 客户区坐标，可处理常见的 100%、125%、150%、175% 和 200% Windows 缩放。选择完成后如果移动或缩放 VRChat、改变系统缩放、旋转屏幕，或把窗口移动到不同缩放比例的显示器，请重新选择区域。VRChat 窗口不要横跨两个显示器。

### 启动 OCR

点击“启动 OCR”。识别到的新文字会进入有界翻译调度器，译文显示在独立浮窗；主窗口诊断表可查看当前会话的识别原文、译文和置信度。

默认的“自动”模式优先使用 Windows Graphics Capture 获取所选窗口内容。普通窗口覆盖在 VRChat 上方时，不应被 OCR 当成 VRChat 画面。目标窗口列表会排除 VRCTranslate 自己的进程，并同时显示窗口标题和进程名。

可在“设置 → OCR → 捕获与识别”选择后端并点击“测试捕获（仅内存预览）”。预览会在设置页显示，切换目标、关闭页面或退出后释放，不会生成图片文件。

只有在窗口捕获不兼容时才选择“MSS 屏幕坐标兼容模式”。该模式截取桌面上当前可见像素，启动或测试时会隐藏主窗口；VRChat 必须保持可见，且识别区不能被其他窗口遮挡。

译文浮窗只显示译文，不显示原文、时间、置信度或 OCR 控件。停止 OCR 后浮窗隐藏，旧会话中尚未返回的译文会被忽略。

### 低延迟规则

- 传统在线服务最多并发 2，本地 Argos 和大模型最多并发 1。
- 队列达到设置上限时直接丢弃新任务，不无限积压。
- 任务超过有效期后不再展示。
- 同一会话的相同文字使用内存缓存，退出后清空。
- 结果按识别顺序展示，不因网络返回顺序而错位。

### 译文浮窗

在“设置 → OCR → OCR 译文浮窗”中可调整：

- OCR 译文置顶。
- 锁定并鼠标穿透。
- 透明度、字号、最大显示条数和单条显示时长。

需要移动或缩放时，先关闭鼠标穿透：拖动浮窗可移动，从右下角拖动可缩放，完成后再锁定。MSS 模式仍建议把浮窗移出识别区。

主窗口不会从 Windows 截图工具中排除，因此正常截图时应当可见。快捷输入和 OCR 译文浮窗仍可能使用系统捕获排除，以避免 MSS 模式重复识别自身内容。

## 6. OCR 隐私与限制

- 捕获帧只在内存中短暂处理，没有截图保存功能或图片写盘路径。
- 画面不会上传；在线翻译接口只收到 OCR 得到的文字。
- 识别原文、译文和会话缓存退出后清除。
- 不建立聊天数据库或历史记录。
- 特殊字体、小字号、透明文字、动态背景和遮挡可能降低准确率。
- Windows 窗口捕获通常不受普通窗口遮挡影响，但窗口最小化、受保护内容或独占全屏仍可能无法返回有效帧；建议使用无边框窗口。

## 7. 便携文件与日志

所有可写数据位于软件目录的 `data`：

| 路径                | 内容                          |
| ------------------- | ----------------------------- |
| `data\config.json`  | 版本 2 设置和明文 API 密钥    |
| `data\logs\app.log` | 轮转状态日志，最多约 4 个文件 |
| `data\cache`        | 下载和第三方临时缓存          |
| `data\models\argos` | Argos 语言模型                |

日志只记录状态、请求编号和错误类别，不记录聊天正文、API 密钥或截图。在“设置 → 便携数据与诊断”可清空运行日志。

移动或备份软件时应处理整个软件目录。删除整个目录即可同时移除配置、缓存和模型。

## 8. 常见问题

### 按 Enter 后 VRChat 没有文字

检查 VRChat 是否启用 OSC、主机是否为 `127.0.0.1`、端口是否为 `9000`，并等待最小发送间隔。程序提示“已提交至 OSC”不等于 VRChat 已显示。

### VRChat 没有出现 `...`

确认 VRChat OSC 已启用并允许 Chatbox。typing 同步在应用中始终开启，不存在设置开关；UDP 不提供回执。

### OCR 翻译太慢

优先给 OCR 选择 DeepL、Google 或 Argos，不建议使用大模型；减小识别区域，并按电脑性能适当增加识别间隔。

### Argos 没有可用模型

确认当前版本包含离线组件，联网刷新模型列表，并安装与路由语言方向完全一致的语言对。组件与模型是两部分，只安装组件不会自动下载模型。

如果目标语言对没有直接模型（如中文 ↔ 日语），页面会提示中转翻译方案。点击"安装中转所需模型"可一键安装所需的全部中间模型，安装后即可自动中转翻译。中转翻译极不可靠，可能出现严重语义偏差或幻觉内容，重要沟通请使用在线翻译服务。

### 软件提示 data 不可写

把完整软件目录移动到 D/E 盘普通目录后再运行。不要放在 `Program Files`，不要直接从压缩包内部启动。程序不会改用 C 盘 AppData。

### 找不到 VRChat 窗口或没有识别结果

点击刷新并确认下拉项的进程名是 VRChat，重新选择更精确的文字区域，再到“设置 → OCR”执行内存测试捕获。Windows 捕获失败时查看明确的后端提示；MSS 模式则必须保持 VRChat 可见且不被遮挡。两个程序尽量使用相同权限级别。

## 9. 安全边界

- 不要分享 `data\config.json`。
- 软件只支持 PC 桌面模式，不支持 PCVR。
- OCR 是屏幕辅助识别，不是直接读取 VRChat 聊天数据。
- 机器翻译和 OCR 都可能出错，重要内容应结合原画面判断。
