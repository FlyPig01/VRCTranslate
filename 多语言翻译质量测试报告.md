# VRCTranslate 多语言翻译质量测试报告

## 1. 测试信息

- 测试日期：2026-07-21
- 开发环境：Windows 10/11、Python 3.11.4
- 测试版本：v0.10.0 开发候选版
- 真实翻译服务：腾讯云机器翻译 `TextTranslate`
- 测试语言：繁体中文、韩语、法语、德语、西班牙语、俄语
- 测试方式：固定短句双向翻译、人工核对语义、语气、信息完整性和 VRChat 口语可用性

本报告中的评分为本次固定样本的人工判断，不代表服务商在全部文本、领域和时间下的统计质量。当前开发配置只有腾讯云机器翻译具备可用凭据，因此 DeepL、Google Cloud、阿里云和大模型档案没有冒用或读取不存在的密钥进行真实对比。

## 2. 固定测试句

中文原文：

> 晚上好，要一起去新的世界玩吗？

反向测试分别使用自然的繁中、韩、法、德、西、俄语表达，再翻译回简体中文。评分采用 5 分制：5 分表示语义、语气和信息均自然完整；3 分表示基本可用但存在明显偏差；1 分表示不建议直接发送。

## 3. 腾讯云真实翻译结果

| 方向 | 实际结果 | 延迟 | 评分 | 判断 |
|---|---|---:|---:|---|
| 简中 → 繁中 | 晚上好，要一起去新的世界玩嗎？ | 419 ms | 5.0 | 繁简转换和语义完整 |
| 繁中 → 简中 | 晚上好，要一起去新的世界玩吗？ | 226 ms | 5.0 | 完整准确 |
| 简中 → 韩语 | 안녕하세요. 같이 새 세계에 놀러 갈까요? | 379 ms | 4.0 | 自然可用；“晚上好”弱化为普通问候 |
| 韩语 → 简中 | 你好，要一起去新世界吗？ | 376 ms | 4.5 | 语义完整，口语自然 |
| 简中 → 法语 | Bonjour, voulez-vous aller jouer dans le nouveau monde ? | 408 ms | 3.0 | 将晚间问候改成普通问候，且弱化“一起” |
| 法语 → 简中 | 晚上好，你想和我一起去一个新世界吗？ | 458 ms | 4.5 | 信息完整，表达自然 |
| 简中 → 德语 | Gute Nacht, möchten Sie mit uns in die neue Welt gehen? | 329 ms | 2.5 | `Gute Nacht` 更接近睡前“晚安”，并把对象改成“和我们” |
| 德语 → 简中 | 晚上好，你愿意和我一起去探索一个新的世界吗？ | 346 ms | 4.5 | 基本完整，“访问”被润色为“探索” |
| 简中 → 西语 | ¿Te gustaría ir a un nuevo mundo? | 354 ms | 2.5 | 丢失问候、“一起”和“玩”的部分语义 |
| 西语 → 简中 | 晚安，呃？你想和我一起去一个新世界吗？ | 471 ms | 2.5 | 插入无来源语气词“呃”，不适合直接发送 |
| 简中 → 俄语 | Добрый вечер, вы собираетесь вместе играть в новый мир? | 409 ms | 4.5 | 语义基本完整，表达可用 |
| 俄语 → 简中 | 晚上好，想一起去看看新世界吗？ | 413 ms | 4.5 | 自然且符合 VRChat 口语 |

延迟统计：平均约 382 ms，中位数约 394 ms，最短 226 ms，最长 471 ms。以上只包含一次 API 请求耗时，不包含 OCR、语音识别、队列等待和界面绘制。

## 4. 质量结论

| 语言 | 当前腾讯云建议 | 说明 |
|---|---|---|
| 繁体中文 | 推荐 | 繁简双向稳定 |
| 韩语 | 推荐 | 样本语义完整，偶有问候语气弱化 |
| 法语 | 可用 | 反向较好，中文转法语存在语气和人称变化 |
| 德语 | 谨慎使用 | 可能混淆“晚上好”和睡前“晚安”，也可能改变人称 |
| 西班牙语 | 不作为默认推荐 | 本次两向测试均出现明显信息偏差 |
| 俄语 | 推荐 | 本次双向样本自然完整 |

建议继续允许所有档案选择这些语言，但不要把“服务支持语言代码”描述成“翻译质量相同”。德语、西班牙语和法语的日常聊天优先建议 DeepL；需要严格保留语气、人称和上下文时，可使用低延迟大模型并保持当前固定翻译提示词。

## 6. 本地 OCR 真实测试

测试图片全部在内存中由系统字体生成，没有保存截图；模型下载到项目 `tmp` 下的临时目录，测试结束后自动清理。

| 自然语言 | 实际模型包 | 模型本体 | 测试结果 | 冷启动识别 |
|---|---|---:|---|---:|
| 韩语 | PP-OCRv5 Korean Mobile | 13,488,748 B | 32px 深色字幕样式完整识别“한국어 테스트” | 约 2.3 s |
| 法语 | PP-OCRv5 Latin Mobile | 7,904,513 B | 完整识别“Bonjour VRChat” | 约 1.1 s |
| 俄语 | PP-OCRv5 Cyrillic Mobile | 8,074,092 B | 完整识别“Привет VRChat” | 约 1.1 s |

韩文测试发现，过大的孤立黑字会被公共检测模型拆成多个单字框并降低准确率；接近 VRChat 字幕的 32px 白字深色背景可以完整识别。因此韩文包可用，但识别区域应尽量贴近实际字幕，不能用超大海报字测试结果替代聊天场景结论。

## 7. 语音语言能力验证

- 本地 SenseVoice 本轮只新增韩语选项，没有把法语、德语、西班牙语或俄语标记为本地支持。
- 腾讯语音源语言按识别引擎过滤，例如 `16k_ja` 只显示日语，`16k_multi_lang` 显示其覆盖的英、日、韩、西、法、德语。
- 阿里云语音档案新增控制台项目语言字段；翻译路由只显示该档案声明的源语言。
- 没有使用合成音频冒充真实玩家语音质量测试。本轮完成的是能力过滤、配置持久化和 UI 测试；韩语语音识别质量仍需要真实韩语游戏语音样本单独评估。

## 8. 最终建议

1. 发布时将多语言功能标为可用，但明确腾讯云在德语和西班牙语样本上的偏差。
2. 默认翻译路由保持“测试回显”，不替用户自动选择付费服务。
3. OCR 模型继续按需下载，不打进基础安装包。
4. 本地语音只新增韩语；其他本地多语种以后单独评估 `whisper.cpp base q5_0`，本版本不增加运行库和模型体积。
5. 后续加入母语社区审校流程，并将固定质量样本扩展到问候、邀请、否定、专有名词和长句五类。
