# VRCTranslate 使用说明

> 平台：Windows 10/11 64 位、VRChat PC 桌面模式  
> 环境安装和启动命令：[README.md](README.md)

本说明对应 `VRCTranslate v0.10.0`。程序不支持 PCVR，也不会读取或修改 VRChat 进程内存。

## 界面语言

设置页顶部使用语言下拉菜单，可在简体中文、繁體中文、English、日本語、한국어、Français、Deutsch、Español、Русский之间切换。语言名称始终使用自身写法，选择后立即保存并刷新全部界面，无需重启。

## 1. 使用前准备

1. 启动 VRChat 桌面模式。
2. 在 VRChat 快捷菜单中启用 OSC。
3. 启动 VRCTranslate。
4. 在“设置 → 翻译模型 → 服务档案”建立真实翻译档案，再到“翻译路由”分别选择自身消息、OCR 和语音识别后的文字翻译服务。
5. 如需使用本地 OCR，进入“设置 → OCR → 本地 OCR 模型”，下载与 OCR 源语言对应的中文、日文、英文、韩文、拉丁文字或西里尔文字模型。多模态独立浮窗不需要本地模型。
6. 如需翻译他人语音，先在“设置 → 语音”安装并验证本地 SenseVoice，或配置腾讯云/阿里云在线档案；再到“设置 → 翻译模型 → 翻译路由 → 他人语音”选择源语言、目标语言、文字翻译档案和术语库开关。

程序启动后会出现主窗口和单行快捷输入浮窗。主窗口负责配置、OCR 和语音字幕控制，永不强制置顶。

程序会在主窗口、设置弹窗及全部悬浮窗中拦截 Tab 和 Shift+Tab，避免误触后改变当前输入焦点；页面标签仍可使用鼠标切换。

首次启动只有“测试回显”档案，接口地址、API 密钥和模型名称均为空。它不会执行翻译；若直接使用，结果会带有测试回显标记。开发者本机的 `config.json` 和 `data\config.json` 不会成为其他用户的默认配置。

### 设置如何保存

- “快捷输入”页的输入浮窗选项会自动保存。
- “OCR 翻译”页的目标程序、悬浮球和译文显示选项会自动保存。
- “语音”页的字幕浮窗选项会自动保存。
- “设置”页中的翻译模型、OSC、OCR 参数和诊断设置需要点击右上角“保存修改”，也可按 `Ctrl+S`。离开时如果仍有修改，程序会询问保存、放弃或取消。

## 2. 快捷输入自己的文字

输入浮窗为空时显示：

> 按Enter发送消息至VRChat

操作方法：

1. 在浮窗中输入文字。
2. 按 Enter。
3. 程序按“自身消息服务”翻译。
4. 翻译成功后按设置格式处理，并自动加入 OSC 发送队列。

浮窗没有翻译、清空、发送按钮，也没有可编辑译文框。翻译成功后，浮窗卡片会显示最近一条的原文和译文预览，方便确认发送内容。翻译失败时原文会恢复到输入框，预览区保持上一条成功结果；消息过长时按设置执行自动拆分、截断或拒绝发送，默认是自动拆分。

### 输入状态

OSC typing 同步始终开启：

- 输入第一个字符时发送 `typing=true`。
- 每次按键刷新活动计时。
- 停止输入约 1.5 秒、按 Enter、清空、隐藏浮窗或退出时发送 `typing=false`。

VRChat 必须已启用 OSC。OSC 使用 UDP，软件只能确认数据已发往本机端口，无法确认 VRChat 最终是否显示消息或 `...`。

### 浮窗操作

- 在主窗口“快捷输入 → 快捷输入浮窗设置”控制置顶和宽度，修改后自动保存。
- 可拖动浮窗边缘空白处改变位置。
- 关闭浮窗只会隐藏；可通过托盘菜单或双击托盘图标再次显示。
- 托盘图标双击默认打开快捷输入。

## 3. 翻译服务档案

每个档案保存一种服务的参数。自身消息、OCR 和语音识别后的文字翻译可以选择不同档案，互不覆盖。

“服务档案”页面按类型展示档案列表。点击“编辑”会打开与新增档案风格一致的独立弹窗；编辑时协议类型和模型服务商保持锁定，避免把档案误改成不兼容类型。档案分组包括：

- 内置工具：测试回显等程序自带能力。
- 机器翻译服务：DeepL、腾讯云机器翻译、阿里云机器翻译、Google Cloud 和 Google 免费翻译。这些服务使用各自的固定 API 或官方 SDK，不属于大模型。
- 大模型服务：文本大模型和具备图片理解能力的多模态大模型，使用 OpenAI Chat Completions 兼容协议。

点击“添加档案”后必须从以下入口之一创建：

- 机器翻译：提供 DeepL、腾讯云、阿里云、Google Cloud 和 Google 免费翻译；腾讯云使用 SecretId/SecretKey，阿里云使用 AccessKey ID/AccessKey Secret。
- 模型服务商：只提供 DeepSeek、通义千问、豆包、MiniMax、Kimi、智谱 GLM 和 OpenAI。软件会填写已知的兼容接口地址，模型名称或接入点 ID 仍由用户按服务商控制台填写，避免内置模型版本过时。
- 自定义兼容：用于其他真正兼容 OpenAI Chat Completions 的文本或多模态服务，必须自行填写完整地址、密钥和模型名称。

模型服务商目录只表示大模型档案的归类和常用地址，不表示所有同厂商 API 都使用同一协议。例如通义千问的 Chat Completions、阿里云 NLS 语音识别和其他专用接口不能共用一个档案。档案创建后不能直接跨类型修改“服务类型”；如果选错了机器翻译或大模型类别，应删除后按正确入口重新添加。

### 测试回显

不执行真实翻译，只返回带标记的原文。适合验证 UI 和 OSC。

### DeepL API

1. 新建或选择档案。
2. 服务类型选择“DeepL API”。
3. 填写 API 密钥。
4. Free 用户可把接口地址留空；其他端点按服务商文档填写。
5. 点击“使用固定短句测试翻译”，成功后保存。

DeepL 延迟低、质量高，适合 OCR。

### Google Cloud Translation

1. 在 Google Cloud 启用 Cloud Translation API 并准备 API 密钥。
2. 服务类型选择“Google Cloud Translation”。
3. 填写 API 密钥；官方默认地址可留空。
4. 测试后保存。

Google 语言覆盖广、延迟低，也适合 OCR。

### Google 翻译（免费）

使用无需 API 密钥的 Google 翻译兼容端点。在服务类型中选择“Google 翻译（免费）”即可使用；该方式不提供官方服务等级保证，稳定性可能随网络或端点策略变化。

注意：国内网络通常无法直连 Google，需要代理才能使用。接口地址可留空；软件会优先使用 JSON 免费端点，遇到限流、拒绝访问或响应格式变化时自动切换到 Google 移动网页备用路径。两条路径都不是有服务等级保证的正式付费 API，如有可用镜像也可自行填写接口地址。

### 腾讯云翻译

使用腾讯云机器翻译 TMT 的文本翻译接口。

1. 在腾讯云控制台开通机器翻译服务。
2. 获取 SecretId 和 SecretKey。
3. 服务类型选择"腾讯云翻译"。
4. 在腾讯云翻译档案中分别填写 SecretId 和 SecretKey。
5. 接口地址可留空（默认使用 tmt.tencentcloudapi.com）。
6. 测试后保存。

腾讯翻译限制 5 次/秒，不支持自动识别源语言，需在翻译路由中明确指定源语言。单次文本需低于 6000 字符。每月有免费额度，超出后按量计费。

### 阿里云机器翻译

使用官方 `alibabacloud-alimt20181012` SDK，可选择通用文本翻译 `TranslateGeneral` 或专业文本翻译 `Translate`。

1. 在阿里云控制台开通机器翻译服务，并准备具有最小所需权限的 AccessKey ID 和 AccessKey Secret。
2. 新增“阿里云机器翻译”档案，填写 AccessKey。凭据会明文保存到便携目录的 `data/config.json`，不要分享该文件。
3. “资源地域”应选择机器翻译资源实际开通或调用的 RegionId，而不是软件根据电脑 IP 猜测的位置。常见公共地域可直接选择，也可以输入阿里云后续新增的地域代码。
4. 软件依据 SDK 1.5.2 的官方地域映射填写端点：杭州使用 `mt.cn-hangzhou.aliyuncs.com`，新加坡使用 `mt.ap-southeast-1.aliyuncs.com`，SDK 当前列出的多数其他公共地域使用 `mt.aliyuncs.com`。
5. 一般用户选择“通用文本翻译”；已经开通专业版并需要其能力时再选择“专业文本翻译”。
6. 自动填写的端点可以手动覆盖，但只能填写主机名或不带路径的 `https://` 地址。留空时由 SDK 根据 RegionId 解析。

阿里云支持源语言自动检测。接口额度、支持语种和计费以所选接口及阿里云账号实际开通情况为准。

### OpenAI 兼容接口

填写兼容 API 根地址、API 密钥和模型名称。此模式更适合自身消息和口语翻译；用于 OCR 时界面会提示延迟风险，并自动限制为单并发、最多两条排队和过期结果丢弃。

### 多模态大模型（支持图片理解）

该名称描述的是模型具备“图片理解”能力，不再使用“仅 OCR”作为模型类型名称。当前软件提供图片的入口是 OCR 框选区域，因此实际使用时仍在“翻译路由 → 他人消息（OCR）”选择该档案；自身消息和语音文字翻译不向模型发送图片，所以不会提供该档案。

- 独立译文浮窗：框选区域在内存中压缩后直接发送给多模态服务，不要求安装本地 OCR 模型。
- 识别区域嵌字或同时显示：本地公共检测模型先定位文字，再把带编号的图片副本发送给服务；需要安装任意一个本地 OCR 包。
- 连续模式默认至少间隔 3 秒，并在画面没有达到变化阈值时跳过调用。接口费用以所选服务商为准。
- 设置页的测试按钮只发送程序内存生成的 `Hello VRChat` 测试图片，不读取屏幕。

选择该档案时界面会显示橙色隐私提示。图片不会写入磁盘，但框选区域会上传到用户配置的云端服务。

## 4. 路由与发送设置

“设置 → 翻译模型 → 翻译路由”包含三组互不依赖的选择，分别放在独立卡片中：

- 自身消息（OSC）：服务、源语言、目标语言、消息格式、超长消息策略、日语罗马音处理方式。
- 他人消息（OCR）：服务、源语言、目标语言、日语罗马音处理方式。
- 他人语音（识别后翻译）：文字翻译服务、语音源语言、翻译目标语言和术语库开关。实时语音服务本身在“设置 → 语音”管理。

### 日语罗马音处理

当源语言为日语或自动检测时，可以选择：

- 关闭：不转假名，原样交给翻译服务。
- 自动判断：有把握是日语罗马音时才转换；英文、URL、用户名和无法确定的内容保持原样。
- 强制转换：把未受保护的拉丁字母按日语罗马音处理，只适合用户确认内容是罗马音的场景。

自身消息和 OCR 独立设置。新安装默认自身消息使用“自动判断”，OCR 使用“关闭”；关闭 OCR 罗马音处理不会关闭 OCR 或翻译。旧配置中已经开启 OCR 罗马音转换的用户会迁移到“自动判断”，不会被升级过程擅自关闭。

转换支持没有空格的连续输入，例如 `watashiwagakuseidesu`。常见问候语会规范为 `こんにちは`、`こんばんは`；纯音译无法判断任意长句中的助词和汉字，因此不能保证恢复成完全正确的日文书写。

### 本地术语库

“设置 → 翻译模型 → 术语库”提供两组内容：

- 默认术语：随软件提供，只能查看、筛选或复制，不能修改和删除。
- 我的术语：可以新增、编辑、删除、导入和导出；与默认术语冲突时优先使用。

开发环境中的默认术语文件位于 `src\vrctranslate\infrastructure\glossary\resources\default_glossary.json`；打包后会作为只读资源放入程序的 `_internal` 目录，不需要也不建议手工修改。用户新增和修改的内容单独保存在程序旁的 `data\glossaries\user_glossary.json`。

术语条目统一用于自身消息、OCR 和识别后的语音文字，不再逐条设置使用范围；需要停用时，可以在对应翻译路由中关闭整套术语库。页面先选择“语言对”，例如“简体中文 ↔ 日语”，同一张表会合并显示两个实际翻译方向，并按语言分列。表格只显示序号、两种语言的词语和分类。

内置默认术语按中英日三语概念维护，程序会自动生成三种语言之间的六个翻译方向。用户新增和编辑术语时仍要选择实际的源语言与目标语言，并且只提供具体语言，不提供“任意语言”；用户条目不会机械生成反向规则，需要反向固定译法时应单独添加。OpenAI 兼容接口只接收当前文字实际命中的结构化术语；DeepL、腾讯、阿里云、Google Cloud 和 Google 免费接口使用本地占位符保护，翻译器损坏标记时自动丢弃异常结果并回退普通翻译。

源语言选择“自动检测”时不会增加语言识别请求。跨语言保持原样的专有名词始终可用；假名等文字系统明确的内容按对应语言匹配；拉丁字母或汉字存在多个不同目标结果时跳过该术语。

实际处理顺序是：

```text
自身输入 / OCR 识别结果
→ 原始输入中的用户术语保护
→ 未保护区域进行罗马音处理
→ 再次匹配默认术语和用户术语
→ 调用翻译服务
→ 校验并还原目标术语
→ OSC 发送或 OCR 译文显示
```

用户术语原子保存到 `data\glossaries\user_glossary.json`，不会写入数据库或云端，也不会记录聊天正文。

自身消息还可设置：

| 项目        | 说明                               |
| ----------- | ---------------------------------- |
| 仅译文      | 只发送翻译结果                     |
| 原文 / 译文 | 同时发送原文和译文                 |
| 译文 / 原文 | 先译文后原文                       |
| 自动拆分    | 超长时按 UTF-16 长度分为多条，推荐 |
| 自动截断    | 只发送限制内部分                   |
| 拒绝发送    | 恢复原文并提示                     |

OSC 默认主机 `127.0.0.1`、端口 `9000`、最小间隔 1.5 秒。普通本机使用通常无需修改。

## 5. OCR 翻译其他玩家文字

VRChat OSC 不提供其他玩家的 Chatbox 原文，因此此功能使用窗口 OCR。

OCR 识别语言由“设置 → 翻译模型 → 翻译路由 → OCR 源语言”决定。软件按自然语言自动选择模型包：简中和繁中使用中文包，日语使用日文包，英语使用英文包，韩语使用韩文包，法语/德语/西班牙语共享拉丁文字包，俄语使用西里尔文字包。用户不需要理解或手动设置内部模型映射。

### 安装本地 OCR 模型

首次识别前，进入“设置 → OCR → 本地 OCR 模型”：

- 中文高精度完整包约 85.8 MiB，使用 PP-OCRv5 Server Rec。
- 日文高精度完整包约 78.2 MiB，使用 PP-OCRv6 Medium Rec。
- 英文模型本体约 7.5 MiB，使用 PP-OCRv5 Mobile Rec；作为首个 OCR 包安装时连同共用组件约需 12.7 MiB。
- 韩文模型本体约 12.9 MiB，使用 PP-OCRv5 Mobile Rec。
- 拉丁文字模型本体约 7.5 MiB，使用 PP-OCRv5 Mobile Rec，供法语、德语和西班牙语共享。
- 西里尔文字模型本体约 7.7 MiB，使用 PP-OCRv5 Mobile Rec，当前用于俄语。

点击“下载并安装”后，界面会显示真实下载大小和百分比，完成下载后切换为校验状态。安装成功后会隐藏进度条和下载按钮，只显示已安装版本、语言模型占用和“删除模型”。下载可以取消，失败时可以重新下载。

正式模型只写入 `data\models\ocr`；`data\cache\ocr-models` 只保存下载中的 `.part` 临时文件，成功、失败或取消后都会清理，不是模型的第二份副本。六个语言包共用约 5.2 MiB 的检测与方向分类组件，界面总占用只计算一次。六个包全部安装后约占 194.5 MiB；只安装所需语言即可。所有文件都位于软件目录，不使用 C 盘用户缓存。

### 选择识别区域

1. 保持 VRChat 窗口可见。
2. 进入“主窗口 → OCR 翻译”。
3. OCR 悬浮球会随程序默认显示，不需要手动打开。
4. 使用 Windows 窗口捕获时，在 OCR 页“目标程序”中选择 VRChat；列表不完整时点击“刷新”。使用 MSS 兼容模式时不需要选择进程。
5. 左键悬浮球打开菜单，选择“框选识别区域”，在 VRChat 文字经常出现的位置拖动；按 Esc 可取消。

框选时使用高对比度准星，区域越小，识别越快且误识别越少。

框选后会出现透明的 OCR 识别框。拖动顶部控制条可移动，拖动边缘或四角可缩放；右上角 `X` 会停止 OCR 并移除当前区域，但保留悬浮球以便重新框选。选区贴近屏幕上边缘时，控制条会自动放到识别框内部，确保按钮不会移出屏幕。OCR 运行期间选区会锁定，需要调整时请先暂停。

选区会从 Qt 界面坐标换算到 VRChat 客户区坐标，可处理常见的 100%、125%、150%、175% 和 200% Windows 缩放。选择完成后如果移动或缩放 VRChat、改变系统缩放、旋转屏幕，或把窗口移动到不同缩放比例的显示器，请重新选择区域。VRChat 窗口不要横跨两个显示器。

### 启动 OCR

单击悬浮球可按当前模式开始或暂停，也可在识别框或悬浮球菜单中直接选择模式。OCR 主页面的状态、模式、语言和翻译服务仅用于展示；Windows 窗口捕获的目标程序可以在 OCR 页面直接选择，MSS 模式则直接捕获屏幕坐标。语言和翻译服务仍在“设置 → 翻译模型 → 翻译路由”中修改。

- 单次识别：只捕获、识别和翻译一次，完成后自动回到空闲，识别框继续保留。使用嵌字时结果不会按秒数自动消失；开始下一次识别、关闭识别框、移动或缩放选区、切换目标程序时才会清除。
- 持续识别：按识别间隔取帧，结合画面变化阈值决定是否执行 OCR；同一识别框中静止区域的文字不会因为其他区域变化而反复翻译。新一轮产生有效译文时才会整体替换上一轮结果。

默认的“自动”模式优先使用 Windows Graphics Capture 获取所选窗口内容。一次 OCR 运行只建立一个持续捕获会话，不会为每一帧反复创建原生捕获对象。普通窗口覆盖在 VRChat 上方时，不应被 OCR 当成 VRChat 画面。目标窗口列表会排除 VRCTranslate 自己的进程，并同时显示窗口标题和进程名。

可在“设置 → OCR → 捕获与识别”选择后端并点击“测试捕获（仅内存预览）”。预览会在设置页显示，切换目标、关闭页面或退出后释放，不会生成图片文件。

只有在窗口捕获不兼容时才选择“MSS 屏幕坐标兼容模式”。该模式直接截取虚拟桌面上的当前可见像素，不绑定也不激活任何目标进程；切换到 MSS 后应通过 OCR 悬浮球重新框选屏幕区域。VRChat 必须保持可见且不能被其他窗口遮挡。程序不再通过周期性隐藏和显示自身窗口完成捕获；无法安全排除时会给出错误提示。

OCR 主页面的“译文显示方式”提供三种选择：

- 独立译文浮窗：保持原有列表式显示，也是旧配置迁移后的默认值。
- 识别区域嵌字：在识别框内容区域内遮盖原文并绘制译文。
- 浮窗与嵌字同时显示：同时保留列表和原文位置提示。

同一组选项也可以从悬浮球左键菜单的“译文显示方式”，或识别框顶部的“译文”菜单修改。三个入口使用同一个配置并实时同步。

嵌字会根据 OCR 原始行框估算位置和字号，逐行遮盖原文，并只按照译文排版后的实际行宽绘制底板，不会再给整个识别区域铺背景。标题、正文段落和每个项目符号会按空间位置拆成独立翻译块，正文内部的正常换行仍作为同一段处理，避免整页文字都从同一个左上角重新排版。较长译文优先使用同一栏内的剩余宽度，垂直方向不会越过下一个原文块；空间仍不足时才缩小字体并限制在当前块内，避免盖住后续项目符号。“嵌字遮盖强度”控制遮盖底板透明度，“嵌字自动对比色”会按画面亮度选择深浅配色。它不会修改或注入 VRChat，只是一个透明、无焦点、鼠标穿透的窗口。目标窗口最小化或失去前台焦点时嵌字会隐藏；系统无法可靠排除嵌字层时会自动回退独立浮窗，避免 OCR 重复识别译文。

嵌字无法获知原字体、描边和字距，也不能无痕恢复动态背景，因此只能尽量接近原位置和字号。旋转、竖排、弧形、强透视文字以及变化剧烈的背景不适合首版嵌字。

独立译文浮窗默认同时显示较小的识别原文和醒目的译文；可在 OCR 主页面关闭“显示识别原文”，此时浮窗只保留译文。浮窗不显示时间、置信度或 OCR 控件。同一轮识别的多条译文会一起保留，直到下一轮产生有效译文；暂停 OCR 不会立即清空当前内容。

### 低延迟规则

- 传统在线服务最多并发 2，大模型使用单并发。
- 同一识别轮次作为一个整体进入有界队列，不会只丢掉这一轮后面的文字块；上一轮仍在翻译时跳过新的旧画面，避免无限积压。
- 任务超过有效期后不再展示。
- 同一会话的相同文字使用内存缓存，退出后清空。
- 持续模式按文字框位置保存上一轮内存快照，只提交新增或改变区域的文字，不使用“重复冷却秒数”。
- 结果按识别顺序展示，不因网络返回顺序而错位。

### 译文浮窗

在“主窗口 → OCR 翻译 → OCR 译文浮窗与位置尺寸”中可调整：

- OCR 译文置顶。
- 锁定并鼠标穿透。
- 是否显示识别原文。
- 背景透明度、字号和最大显示条数。透明度只改变浮窗背景，文字保持清晰。
- 原文、译文和拖动条始终带有独立的深色高对比度衬底；即使外框透明度较低，在白色或彩色游戏画面上也不会直接使用浅色文字裸显。
- “显示译文浮窗”可在没有新识别结果时检查外观；“重置位置和尺寸”会恢复默认大小并放回主屏幕右上区域。下方摘要会显示当前坐标和尺寸。

需要移动或缩放时，先关闭鼠标穿透：按住浮窗顶部专用拖动条移动，从右下角拖动缩放，完成后再锁定。位置、尺寸和 OCR 页面中的浮窗设置都会自动保存。MSS 模式仍建议把浮窗移出识别区。

主窗口不会从 Windows 截图工具中排除，因此正常截图时应当可见。快捷输入和 OCR 译文浮窗仍可能使用系统捕获排除，以避免 MSS 模式重复识别自身内容。

## 6. 处理其他玩家语音

此功能永久限定 Windows PC 桌面使用，不发展 VR 模式。它不会采集自己的麦克风，也不会把语音或译文通过 OSC 发送。

1. 进入“设置 → 语音”，选择本地或在线识别。默认没有语音档案，程序不会替用户自动创建或启用服务。
2. 本地识别：先在“本地语音识别组件”卡片下载并安装 SenseVoiceSmall INT8，再新增“本地 · SenseVoiceSmall INT8”档案。下载量约 246.3 MiB，安装后约占 255～256 MiB，下载包与安装暂存短暂共存时需约 533 MiB 可用空间；模型、运行库和缓存全部位于软件目录的 `data`，不写入主虚拟环境或 C 盘用户目录。安装开始时会并行读取候选源最多 1 MiB 的小样本并自动选择实际吞吐量更高的来源，首选源失败后自动换源；测速数据只经过内存，不留下测速文件。
3. 在线识别：腾讯云填写 AppID、SecretId、SecretKey 和识别引擎；其中 `16k_multi_lang` 可自动识别英语、日语、韩语等 15 种语言，但不包含中文，中文与英语混合内容应选择 `16k_zh_en`。阿里云填写智能语音 AppKey、AccessKey ID 和 AccessKey Secret，软件会在内存中换取短期 NLS Token。两者使用官方专用协议，不接受所谓“OpenAI 兼容语音地址”。
4. 在档案行点击“待验证”。在线档案会验证鉴权、模型和实时连接，可能产生少量费用；本地档案会校验模型 SHA-256、运行库并实际加载，不产生接口费用。模型被手工删除或损坏时，即使档案曾验证通过，启动前仍会重新检查并明确报错。
5. 点击档案行即可把它设为当前档案，当前行通过背景高亮表示，不再显示“当前使用”文字或“使用”按钮。每行仅保留验证状态、编辑和删除三个操作。
6. 进入“设置 → 翻译模型 → 翻译路由 → 他人语音（识别后翻译）”，选择语音源语言、翻译目标语言、文字翻译档案和术语库。语音识别只返回原文，翻译固定交给这里选择的文字翻译档案。
7. 返回主界面“语音”，点击“刷新程序”，选择 VRChat；测试时也可以选择 VLC、浏览器或其他正在播放声音的程序。
8. 点击语音悬浮球展开字幕窗，再点击右上角的播放图标开始识别。在线档案会持续推送内存 PCM 并显示中间结果；本地档案在检测到约 650 毫秒句末静音后，只把完整内存片段识别一次。两种方式都只在一句话最终确定后请求一次文字翻译。
9. 在语音主页直接调整字幕浮窗置顶、显示译文/原文/两者、透明度、字号和最大条数；展开字幕窗后还可拖动右下角缩放手柄调整宽高。新增实时原文或译文时列表会自动跳转到最新条目，样式与尺寸都会自动保存，不在全局设置页重复出现。
10. 点击字幕窗右上角的暂停图标会立即结束捕获和识别；收起字幕窗也会自动暂停并恢复为悬浮球，待处理的旧分段和迟到事件不会继续显示。音频片段只存在于内存，不保存录音。

删除本地组件前程序会先释放模型。若 Windows 仍占用已经加载的原生 DLL，卡片会显示“等待重启后删除”；正常退出并重新启动一次 VRCTranslate 后会自动删除剩余运行库，再次安装前必须先完成这次重启清理。

“设置 → 语音 → 人声检测”包含三个本地预筛选参数：

| 设置 | 默认值 | 含义 |
| --- | ---: | --- |
| 人声触发阈值 | 350 | 按 PCM 平均振幅判断是否可能有人声。数值越低越灵敏；音乐或噪声经常触发时提高，漏掉轻声时降低。 |
| 静音结束时间 | 650 毫秒 | 在线模式用于回到等待状态；本地模式以这段连续静音判定一句话结束并提交识别。太短会切断自然停顿，太长会延后本地字幕。 |
| 最短连续人声 | 300 毫秒 | 声音连续超过阈值多久才确认为人声。太短容易被短促噪声触发，太长会推迟字幕。触发前的短暂音频只保留在内存，因此不会直接丢掉句首。 |

最长音频段使用内部 12 秒安全上限，防止长时间背景声无限占用内存，不作为普通用户设置项。

语音功能要求 Windows 10 2004（内部版本 19041）或更高版本。不支持进程捕获时会停止并报错，不会改为麦克风或整个系统输出。本地 SenseVoice 完全离线识别；腾讯云和阿里云接收内存 PCM。VRCTranslate 不在本地保存音频、原文、译文或字幕历史，在线服务商是否留存请求以其条款为准。

目标程序自己的背景音乐、视频和环境音仍可能被 ASR 当成语音。多人同时说话时也可能混句，首版不提供说话人分离或玩家身份标注。

## 7. OCR 隐私与限制

- 捕获帧只在内存中短暂处理，没有截图保存功能或图片写盘路径。
- 使用本地 OCR 路由时，在线翻译接口只收到 OCR 得到的文字，画面不会上传。
- 只有用户主动选择“多模态大模型（图片理解）”档案时，框选区域图片才会发送到用户配置的云端服务；图片仍不会保存到本地。
- 识别原文、译文和会话缓存退出后清除。
- 不建立聊天数据库或历史记录。
- 特殊字体、小字号、透明文字、动态背景和遮挡可能降低准确率。
- Windows 窗口捕获通常不受普通窗口遮挡影响，但窗口最小化、受保护内容或独占全屏仍可能无法返回有效帧；建议使用无边框窗口。

| 路径 | 内容 |

移动或备份软件时应处理整个软件目录。删除整个目录即可同时移除配置和缓存。

## 9. 常见问题

### 按 Enter 后 VRChat 没有文字

检查 VRChat 是否启用 OSC、主机是否为 `127.0.0.1`、端口是否为 `9000`，并等待最小发送间隔。程序提示“已提交至 OSC”不等于 VRChat 已显示。

### VRChat 没有出现 `...`

确认 VRChat OSC 已启用并允许 Chatbox。typing 同步在应用中始终开启，不存在设置开关；UDP 不提供回执。

### OCR 翻译太慢

优先给 OCR 选择 DeepL、Google、腾讯或阿里云机器翻译，不建议使用大模型；减小识别区域，并按电脑性能适当增加识别间隔。

### 语音识别没有结果

确认目标程序正在播放清晰人声，且选择的是实际发声的进程。浏览器和播放器可能使用子进程，本程序会包含所选进程树；如果程序重启或出现多个同名窗口，请刷新后重新选择。随后检查当前高亮档案是否已验证。本地档案还应确认组件卡片显示“已安装”，必要时点击“校验组件”；在线档案检查网络、额度和服务配置。最后检查统一翻译路由中的文字翻译档案是否可用。

### 软件提示 data 不可写

把完整软件目录移动到 D/E 盘普通目录后再运行。不要放在 `Program Files`，不要直接从压缩包内部启动。程序不会改用 C 盘 AppData。

### 找不到 VRChat 窗口或没有识别结果

先确认“设置 → OCR → 本地 OCR 模型”中已安装与 OCR 源语言一致的模型。Windows 窗口捕获模式下，在 OCR 页重新选择进程名为 VRChat 的目标程序；MSS 模式不选进程。随后通过悬浮球重新框选更精确的文字区域，最后到“设置 → OCR”执行内存测试捕获。Windows 捕获失败时查看明确的后端提示；MSS 模式则必须保持 VRChat 可见且不被遮挡。两个程序尽量使用相同权限级别。

### 更新后界面仍像旧版本

先从托盘菜单彻底退出 VRCTranslate，并在任务管理器确认旧进程已经结束，再启动新目录中的程序。发布版必须保留 exe 同级的 `_internal` 目录，不能只用新 exe 覆盖旧版本。

## 10. 安全边界

- 不要分享 `data\config.json`。
- 软件只支持 PC 桌面模式，不支持 PCVR。
- OCR 是屏幕辅助识别，不是直接读取 VRChat 聊天数据。
- 机器翻译和 OCR 都可能出错，重要内容应结合原画面判断。
